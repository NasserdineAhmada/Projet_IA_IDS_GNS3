\section{Descriptive statistics and outliers}

\texttt{df.describe()} reveals two important anomalies:

\begin{itemize}[leftmargin=*,itemsep=2pt]
    \item \textbf{Inconsistent negative values:} \texttt{Port\_source} (min $-5000$), \texttt{Port\_destination} (min $-4980$), \texttt{Nb\_paquets\_envoyés} (min $-3200$), \texttt{Tentatives\_connexion} (min $-19860$) — physically impossible, likely due to data generation or extraction errors.
    \item \textbf{Extreme values:} e.g. \texttt{Nb\_flag\_ACK} — Q3 = 130 but max $\approx$ 4.7 million.
\end{itemize}

In an anomaly detection context, such extreme values may correspond to real attacks (flooding, brute force). Therefore, they are not systematically removed (see Part 2).

Most columns contain between 2\% and 8\% missing values, relatively evenly distributed (max: \texttt{Protocole} 8,184, \texttt{Service\_cible} 8,016). \section{Summary}

\begin{itemize}[leftmargin=*,itemsep=2pt]
    \item large dataset (103k rows), suitable for machine learning;
    \item moderate and homogeneous missing values $\rightarrow$ imputation is feasible;
    \item incorrectly typed numerical columns (\texttt{\#NOM?}) $\rightarrow$ requires conversion;
    \item useless column \texttt{0} and categorical variables (\texttt{Protocole}, \texttt{Service\_cible}) require encoding;
    \item moderate class imbalance and potentially informative outliers.
\end{itemize}

These observations directly guide the preprocessing pipeline (Part 2).
